注意 图片真实crc32.txt 这一文件，其文件大小为6，但十六进制形式的CRC32值应该有8位。考虑一种可能性，即PNG图片IHDR块的真实CRC校验位（在此暂不考虑PNG内其它数据块如IDAT块等的CRC校验位）和该txt文件的CRC32值是一致的，均为0x59F1D4BE，尝试在此基础上爆破图片的宽和高：
import zlib
import struct

# 同时爆破宽度和高度
filename = 'C:/Users/Administrator/Desktop/file.png'
with open(filename, 'rb') as f:
    all_b = f.read()
    data = bytearray(all_b[12:29])
    n = 4095
    for w in range(n):
        width = bytearray(struct.pack('>i', w))
        for h in range(n):
            height = bytearray(struct.pack('>i', h))
            for x in range(4):
                data[x+4] = width[x]
                data[x+8] = height[x]
            crc32result = zlib.crc32(data)
            if crc32result == 0x59F1D4BE:
                print("宽为：", end = '')
                print(width, end = ' ')
                print(int.from_bytes(width, byteorder='big'))
                print("高为：", end = '')
                print(height, end = ' ')
                print(int.from_bytes(height, byteorder='big'))

那么就看一下图片：
这是J.R.R.托尔金在其作品《魔戒》中创造的Tengwar script书写系统所使用的字母，又名Fëanorian字母，可以用于书写昆雅语（Quenya）或辛达林语（Sindarin），但书写规则有所不同（主要区别在于元音写在前一个还是后一个辅音的上方）。昆雅式的Tengwar script书写规则和字母表可以参见Quenya language and the Tengwar script。另一方面，古精灵语（Primitive Quendian）在设定中则是各种精灵语（包括昆雅语和辛达林语在内）的祖语，是最初的精灵所使用的语言，暂时没有找到具体的语法与书写系统设定，很可能托尔金并未给出详细设计。
而本题的图片和上述背景信息没有任何关系，可能是由于百度百科的误导，将一张用Tengwar script来转写英文字母的对应表误认为古精灵语，从而制作的替换密码。该对应表的直接出处之一是豆瓣小站中土红皮书里一个关于用Tengwar script转写英文名的教程。可以看出制图时用的字体也是直接来源于该对应表。此外，网络上也流传有另一张内容相似的对应表。
因此这里反而应该完全忽略古精灵语/昆雅语/Tengwar script等，将其作为简单替换密码，直接按照对应表来解密，得到：
